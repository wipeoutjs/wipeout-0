Using templates

Wipeout view models alone do not have an appearance, for this we need templates. At it's most basic, a view model inherits from the wo.view class and the view class has a templateId property.
Template id's reference the id of a html <script></script> tag with an xml template. It is not strictly html, as

  1. The content must be valid, strict XML.
  2. The content can contain wipeout view models which are not html tags.

In most cases, you will not want the the template of a view model to change. For this case you can set the template id in the constructor of the view model. var myViewModel = wo.view.extend(function () { this._super('MyTemplateId'); // more constructor logic });
