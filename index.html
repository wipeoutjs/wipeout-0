What is Wipeout?

Wipeout is a template engine for knockout. It leverages the superb client binding and templating capabilities of knockout, but goes one step further, introducing the basic functionalities and development patterns of XAML in a HTML/javascript environment.

What does it look like?

Wipeout is built to look as much like WPF and Silverlight as possible. You define components in javascript, give them html templates and nest them like with XAML. Have a look:

A collapsible component

View model class

// create a namespace for our application
window.app = {};

// inherit from the wo.view class
app.collapsible = wo.view.extend({
    constructor: function() {
        // call the base constructor with the default template id.
        this._super("app.collapsible");

        // create a property to set the html template
        // of the collapsible content
        this.contentTemplate = ko.observable("");
    },                    
    // a function which will show or hide the collapsible content
    toggleContent: function() {
        var element = this.templateItems.collapsibleElement;
        $(element).slideToggle("fast");
    }
});
View model Template
(<script type="text/xml" id="app.collapsible">...</script>) 
<div data-bind="click: toggleContent"
    class="panel-heading"
    style="cursor: pointer">
    <h3 class="panel-title"
        data-bind="text: model().header"></h3>
</div>
<div class="panel-body">
    <div id="collapsibleElement">
        <!-- Inbuilt wipeout component to display html -->    
        <wo.contentControl template="contentTemplate"
            woInvisible="true">
        </wo.contentControl>
    </div>
</div>




















Root application component

View model class

// inherit from the wo.view class
app.rootApplication = wo.view.extend({
    constructor: function () {
        // call the base constructor with the default template id.
        this._super("app.rootApplication");
    }
});
View model Template
(<script type="text/xml" id="app.rootApplication">...</script>) 
<div style="width: 300px; margin: 50px"
     class="panel panel-primary">
    <h4 data-bind="text: model().applicationName"></h4>
    <app.collapsible model="$parent.model().collapsibleContent">
        <!-- remember this is the property defined in the 
            app.collapsible constructor. We are setting it here-->
        <contentTemplate>
            <div data-bind="text: model().content">
            </div>
        </contentTemplate>
    </app.collapsible>
</div>

Application definition

Model and setup 
var rootApp = document.getElementById("wipeoutApp");

    applicationName: "Wipeout demo",
    collapsibleContent: {

    }

So Why?

  • Custom Html elements

    Wipeout view models are used in the same way as standard html. So you can have standard <div></div> and custom <shoppingCart /> tags side by side.

  • More componentization than you can handle!

    In wipeout, everything is a component. From your root application to a visual component (like a collapsible) to a business component (like a shopping cart). This makes large complicated pieces of code quite pluggable.

  • Leverage the knockout you know and love

    Wipeout is a template engine for knockout and so adds to it. Excluding 1 or 2 control flow bindings, all of knockout is available if using wipeout. The missing content is either un necessary in the wipeout paradigm or replaced

  • Strict seperation of concerns

    Wipeout is rigidly seperated into 3 areas:

    1. View models. View models are javascript objects which contain view logic and view data. View models inherit from "wo.view" and are mostly declared and created inline in html.
    2. Models. Models are javascript objects which contain business logic and business data. Each view model has a model property.
    3. Templates. Templates are html. Each view model has a "templateId" property to give it appearence. Templates can bind to model and view model properties.

    Do you want to hack some code to gether to something quick and dirty? First you have to decide where to put the hack, then you need to decide which event it should fire off, and before you know it your hack isn't a hack anymore!

  • The framework that does not control

    Wipeout applications are self contained within a single html element. This is great for those who do not want to relinquish all control to a framework or those with a fear of the SPA.
    Plain old vanilla knockout and wipeout can also be used side by side.

  • Your old friend jQuery is welcome

    Although, you will find that jQuery is not as relevant in the MVVM universe as it used to be.
